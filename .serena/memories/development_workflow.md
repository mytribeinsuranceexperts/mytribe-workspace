# Development Workflow

## Standard Process: EXPLORE → PLAN → CODE → COMMIT

### 1. EXPLORE
- Read code using Serena's semantic tools
- Search codebase for relevant symbols
- Review existing tests
- **No coding** in this phase

### 2. PLAN
- Design solution architecture
- Identify edge cases and potential issues
- Get user approval before proceeding
- Use TodoWrite tool for complex multi-step tasks

### 3. CODE
- **TDD when applicable**: Write tests first
- Incremental implementation
- Use Serena's symbol-based editing for precision
- Commit frequently with meaningful messages

### 4. COMMIT
- Follow conventional commit format
- Run tests before committing
- Check git status and review changes
- Push to remote repository

## Best Practices
- **MCP Tools First**: Always use MCP tools (mcp__*) over CLI when available
- **Latest Dependencies**: Use current stable versions, note breaking changes
- **Communication**: Concise, bulleted, factual. No emojis unless requested
- **Token Efficiency**: Use Serena's semantic tools instead of reading entire files
